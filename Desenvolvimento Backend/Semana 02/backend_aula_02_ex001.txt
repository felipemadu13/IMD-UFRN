O Event Loop é um mecanismo do Node.js que lida com a execução de blocos de código. Ele monitora as requisições e as organiza de maneira apropriada. As operações padrão são executadas dentro do Event Loop, mas as operações de entrada/saída são executadas em threads paralelas gerenciadas pelo Node.js. Quando estas operações são encerradas, o Node.js envia uma resposta através de um callback, que é tratado dentro do Event Loop.
A principal vantagem deste mecanismo é, do ponto de vista do programador, tratar o programa como uma única thread, evitando a necessidade de lidar com concorrência e sincronia. Entre as desvantagens está um consumo maior de memória pelo uso de callbacks e Event Emitters, quando está lidando com grandes quantidades de dados.
